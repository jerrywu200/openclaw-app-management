## 🔴 子Agent上下文继承问题（2026-03-30 12:48）

### 问题发现
用户指出：子Agent执行个股分析框架时**没有继承主Agent的记忆和执行要点**，导致分析质量不高。

### 根本原因
子Agent通过`sessions_spawn`创建时：
- ✅ 继承workspace目录
- ❌ 不继承主session对话上下文
- ❌ 不自动注入MEMORY.md、核心教训等

### 影响范围
| 缺失内容 | 影响 |
|----------|------|
| MEMORY.md核心教训 | PE(TTM)计算错误、数据源优先级等 |
| 数据源使用规范 | stock_db_tool正确导入方式 |
| 分析框架版本 | 可能用了旧版本 |
| 持仓信息 | 不知道股票是否在持仓中 |
| 缓存机制 | 没有检查已有缓存 |

### 解决方案思路

**方案A：避免使用子Agent执行复杂分析**
- 个股分析框架太复杂，依赖大量上下文
- 应该在主session中直接执行

**方案B：在任务描述中显式注入关键信息**
```python
sessions_spawn(
  task="""
  执行洛阳钼业V3.5分析框架。
  
  【关键教训】
  1. PE(TTM)必须用TTM EPS计算，禁止用季度累计EPS
  2. 数据源优先级：stock_db_tool > akshare > web_search
  3. stock_db导入：sys.path.insert(0, '/Users/jw/.openclaw/workspace')
  
  【框架路径】
  - 框架文件：/Users/jw/.openclaw/workspace/stock_analysis_framework.md
  - MEMORY.md：/Users/jw/.openclaw/workspace/MEMORY.md
  """,
  ...
)
```

**方案C：OpenClaw机制改进（需官方支持）**
- 增加子Agent上下文继承选项
- 自动注入workspace中的关键文件

### 当前决策
**个股分析框架不应使用子Agent执行**，应在主session中直接运行。
