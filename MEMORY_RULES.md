# 记忆写入决策框架

> 判断什么内容应该写入长期记忆（MEMORY.md）

---

## 一、写入原则

### ✅ 应该写入长期记忆的内容

| 类型 | 示例 | 原因 |
|------|------|------|
| **用户身份信息** | 姓名、职业、偏好 | 跨session需要知道 |
| **关键决策** | "希望我成为股票分析师" | 影响后续行为 |
| **重要偏好** | "用飞书沟通" | 避免反复询问 |
| **能力配置** | 已安装的skills | 避免重复配置 |
| **里程碑事件** | 第一次完成某项任务 | 记录进展 |
| **用户明确要求记住的** | "记住这个..." | 用户主动要求 |
| **错误教训** | 犯过的错误 | 避免重复 |
| **长期有效的知识** | 投资框架、分析方法 | 持续使用 |
| **成功的执行过程** | 股票分析方法论 | 可复用的能力 |

### ❌ 不应写入长期记忆的内容

| 类型 | 示例 | 原因 |
|------|------|------|
| **临时性信息** | 今日天气、股价 | 会过时 |
| **敏感信息** | 密码、身份证号 | 安全考虑 |
| **细节性对话** | 具体对话内容 | 记忆过载 |
| **可快速获取的信息** | 公开数据 | 浪费记忆空间 |
| **不确定的信息** | 猜测、假设 | 可能错误 |

## 二、决策流程

```
收到信息 → 判断是否重要
                ↓
        ┌───────┴───────┐
        ↓               ↓
    重要性高         重要性低
        ↓               ↓
    是否持久？       写入日常记录
        ↓           (memory/YYYY-MM-DD.md)
    ┌───┴───┐
    ↓       ↓
    是      否
    ↓       ↓
写入      只写入
MEMORY.md  日常记录
```

### 判断问题清单

**问自己这些问题：**

1. **持久性**：这个信息一个月后还有用吗？
2. **重复性**：用户会不会再次提到这个？
3. **影响度**：这会影响我后续的行为吗？
4. **独特性**：这是用户特有的信息吗？
5. **明确性**：用户明确表示要记住吗？

**评分规则：**
- 3个以上"是" → 写入MEMORY.md
- 1-2个"是" → 写入memory/YYYY-MM-DD.md
- 0个"是" → 不记录

## 三、记忆层级

```
┌─────────────────────────────────────────────┐
│  MEMORY.md（长期记忆）                       │
│  - 用户身份、偏好                           │
│  - 核心决策、里程碑                         │
│  - 持久有效的知识                           │
└─────────────────────────────────────────────┘
                    ↑
              提炼总结（每几天）
                    ↑
┌─────────────────────────────────────────────┐
│  memory/YYYY-MM-DD.md（日常记录）            │
│  - 今日完成的工作                           │
│  - 重要对话摘要                             │
│  - 待办事项                                 │
└─────────────────────────────────────────────┘
```

## 四、具体示例

### 示例1：用户说"我希望你成为股票分析师"

```
判断：
- 持久性：✅ 是，长期身份定位
- 重复性：✅ 是，会反复涉及
- 影响度：✅ 是，决定我的行为方向
- 独特性：✅ 是，用户特有需求
- 明确性：✅ 是，明确表达

结论：写入MEMORY.md
```

### 示例2：股票分析任务

**分析过程（方法论）：**
```
判断：
- 持久性：✅ 是，分析方法可复用
- 重复性：✅ 是，每次分析都用
- 影响度：✅ 是，决定分析质量
- 独特性：✅ 是，我的专业能力
- 明确性：✅ 是，用户指定

结论：写入MEMORY.md（长期记忆）
```

**分析结果（具体数据）：**
```
判断：
- 持久性：❌ 否，股价会变化
- 重复性：⚠️ 有参考价值
- 影响度：⚠️ 短期影响
- 独特性：❌ 否，公开数据
- 明确性：❌ 否

结论：写入memory/YYYY-MM-DD.md（日常记录）
```

### 示例3：用户说"记住我的电话是138xxxx"

```
判断：
- 持久性：✅ 是
- 重复性：✅ 是
- 影响度：⚠️ 可能
- 独特性：✅ 是
- 明确性：✅ 是，明确要求记住

结论：写入MEMORY.md（但注意敏感信息处理）
```

## 五、更新频率

| 文件 | 更新时机 |
|------|----------|
| MEMORY.md | 重大事件发生时 + Heartbeat时审查 |
| memory/YYYY-MM-DD.md | 每日结束时总结 |

---

## 六、敏感信息处理

**原则：** 敏感信息不直接写入明文记忆

| 敏感类型 | 处理方式 |
|----------|----------|
| 密码 | 不记录 |
| 身份证号 | 部分脱敏后记录 |
| 银行账户 | 不记录 |
| 私密对话 | 不记录具体内容，只记录"涉及私密话题" |
| 合同细节 | 不记录敏感条款 |
