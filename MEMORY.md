## 核心教训汇总

### 🔴 年报优先使用（2026-03-29 发现）

**问题**：分析时使用季报数据，但数据库已有年报数据

**错误原因**：
```python
# 错误做法 ❌
latest = finance.iloc[0]  # 取最新报告期，可能是季报

# 正确做法 ✅
annual = finance[finance['报告期'].str.contains('年报')]
if len(annual) > 0:
    latest = annual.iloc[0]  # 优先使用年报
else:
    latest = finance.iloc[0]  # 无年报才用季报
```

**年报 vs 季报**：
| 年报 | 季报 |
|------|------|
| ✅ 完整审计 | ❌ 未审计 |
| ✅ 数据完整 | ⚠️ 部分数据缺失 |
| ✅ 年度指标准确 | ⚠️ 累计值需计算 |

**⚠️ 核心教训**：财务分析必须优先使用年报数据！年报比季报更完整、更权威。

### 🔴 PE(TTM)计算错误（2026-03-29 发现并修正）

**问题**：估值评分中使用季度累计EPS而非TTM EPS计算PE

**错误原因**：
```python
# 错误代码
eps = latest_fin.get('基本每股收益(元)', 0)  # 季度累计值！
pe = latest_price / eps  # ❌
```

**正确公式**：
```
TTM EPS = 最新累计EPS + 去年年报EPS - 去年同期累计EPS
PE(TTM) = 股价 / TTM EPS
```

**示例（中际旭创）**：
- 季度累计EPS：6.48元 → PE 92.3倍（错误）
- TTM EPS：7.78元（6.48+4.72-3.42）→ PE 76.86倍（正确）
- 误差：15.4倍（20%）

**修正措施**：
- ✅ 修改`calculate_valuation_score()`函数
- ✅ 增加`get_stock_finance(limit=10)`获取更多期数据
- ✅ 实现TTM EPS动态计算
- ✅ 增加pe_ttm字段到返回值

**⚠️ 核心教训**：PE(TTM)必须使用TTM EPS计算，严禁用季度累计EPS替代！
