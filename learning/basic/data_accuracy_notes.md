# 数据准确性改进方案

> 背景: 英维克分析中PE计算错误（73倍 vs 实际189倍）

---

## 一、错误复盘

### 错误链条

```
估算总股本(3.38亿) → 计算PE(73倍) → 得出"偏高但可接受"结论
      ↓
实际总股本(9.77亿) → PE应为189倍 → 估值显著偏高
```

### 根本原因分析

| 层面 | 问题 | 影响 |
|------|------|------|
| **数据源** | 未获取官方股本数据 | 计算基础错误 |
| **流程** | 估算值未验证就使用 | 错误传导至结论 |
| **意识** | 未考虑股本变动（转增） | 数据过时 |
| **验证** | 无交叉验证机制 | 错误未被及时发现 |

---

## 二、数据源改进方案

### 核心原则

**永远不使用估算值作为计算基础**

### A股数据获取优先级（修订版）

| 数据类型 | 首选源 | 函数/方法 | 备选源 |
|----------|--------|-----------|--------|
| **股本结构** | 同花顺 | `stock_share_change_cninfo()` | web_search |
| **K线历史** | 腾讯 | `stock_zh_a_hist_tx()` | — |
| **实时行情** | 腾讯 | `stock_zh_a_tick_tx_js()` | — |
| **财务报告** | 新浪 | `stock_financial_report_sina()` | — |
| **估值指标** | web_search | 搜索理杏仁/同花顺 | — |
| **实时估值** | 东方财富 | `stock_zh_a_spot_em()` | ⚠️ 成功率低，仅作备选 |

## 三、计算流程改进

### 原流程（有问题）

```
1. 获取K线数据 → 股价 ✓
2. 获取财务数据 → 净利润 ✓
3. 估算总股本 → 总股本 ✗ (估算值)
4. 计算PE
```

### 改进流程

```
1. 获取K线数据 → 股价 ✓
2. 获取财务数据 → 净利润 ✓
3. 获取股本结构 → 总股本 ✓ (官方数据)
4. 检查股本变动 → 是否有转增/配股 ✓
5. 计算PE
6. 交叉验证 → 与理杏仁/东方财富对比 ✓
```

### 新增验证检查点

```python
def validate_valuation_data(stock_code, price, net_profit, pe_calculated):
    """估值数据验证函数"""

    # 检查1: 股本来源
    print(f"[检查] 股本数据来源: {股本数据来源}")
    assert 股本数据来源 in ['同花顺', '东方财富', '公告'], "股本数据未验证"

    # 检查2: PE合理性
    print(f"[检查] 计算PE: {pe_calculated}")
    if pe_calculated > 100:
        print(f"[警告] PE超过100，需确认是否正确")

    # 检查3: 交叉验证
    pe_external = get_external_pe(stock_code)  # 从理杏仁/东方财富获取
    diff = abs(pe_calculated - pe_external) / pe_external
    if diff > 0.2:
        print(f"[错误] PE计算值与外部数据差异{diff*100:.1f}%，需核查")
        return False

    return True
```

## 七、本次错误教训

### 教训总结

1. **估算值是危险的** — 必须用官方数据
2. **股本会变动** — 转增、配股、增发都会改变计算基数
3. **交叉验证是必须的** — 计算结果要与权威来源对比
4. **异常值要警惕** — PE 73倍本身就应该触发怀疑

### 改进承诺

- 所有估值计算前，强制获取官方股本数据
- 计算结果必须与理杏仁/东方财富交叉验证
- 发现差异>20%时，暂停分析并核查
